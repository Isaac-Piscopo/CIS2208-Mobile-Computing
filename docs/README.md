# Documentation Index

This directory contains the supporting documentation for the WiFiNote project.
It includes the final project report and assets for the main repository readme.

## Directory Contents

* `WiFiNote_report.docx`: The final project report for the submission.
* `screenshots/`: A collection of application images used to illustrate features in the root README file.

## Relationship between Report and Code

The report documents the design and the user-centred design process.
It explains the research and the reasoning behind specific feature choices.
The code in the repository is the functional implementation of these designs.
The application behaviour follows the specifications described in the report.

## Report Build Tooling

The report is generated using a local script named `build_report.py`.
This tool remains outside the repository by design.
It is a local build utility and does not form part of the submitted codebase.
The repository only tracks the final document output.
